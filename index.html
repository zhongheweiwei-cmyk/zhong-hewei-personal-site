How I Work

我的工程工作方式

01

把业务语言翻译成工程边界

先明确用户、使用场景、数据来源、安全边界和验收标准，再拆成前端、后端、数据库、自动化流程和 LLM 调用链。

02

用结构化合约约束大模型

通过 JSON Schema、Prompt 合约、内容校验和反馈回流，降低 LLM 输出不可控带来的工程风险。

03

为真实运行环境设计降级路径

行情缺失、GPT 失败、经济日历不可用、ASR 失败、UI 前台焦点异常等场景，都需要本地规则兜底和清晰状态提示。

04

交付能被接手的系统

输出运行手册、接口说明、交接文档、验证脚本和安全边界，让项目不仅能跑，也能维护。
